# Gap Capture

Use this file when the answer is blocked by missing or weak source material and you want to gather reusable context efficiently.

## Smallest Useful Ask

Request the smallest durable artifact that can improve future answers, such as:

- a repo path
- a runbook path
- a doc repo path
- a sample command
- an environment file name
- a screenshot of the relevant doc section
- a list of canonical doc locations

## What to Capture for This Skill Over Time

Good future reference material for this skill includes:

- authoritative repo list by tool family
- canonical doc or runbook locations
- owner or maintainer map
- environment naming conventions
- runner and host naming patterns
- common acronyms and internal terms
- "use this, not that" guidance between overlapping tools

## Safe Placeholder Pattern

When adding new reference content, prefer:

- confirmed facts
- known gaps
- where to verify

Avoid storing:

- passwords or secrets
- fragile one-off personal notes
- claims that are only remembered, not verified

## User Prompt Pattern

When you need help from the user, ask for one compact thing like:

- "Point me at the repo or runbook you trust most for this flow."
- "Send the doc path or shared repo location you use for this tool."
- "Which environment name or runner name should I anchor on?"
