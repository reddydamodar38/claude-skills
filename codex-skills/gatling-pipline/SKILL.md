# Gatling Pipline

Execute the four skills in this exact sequence:
1. [$gatling-converter](C:\Users\prakash\.codex\skills\gatling-converter\SKILL.md)
2. [$gatling-runner](C:\Users\prakash\.codex\skills\gatling-runner\SKILL.md) for validation run
3. [$gatling-fixer](C:\Users\prakash\.codex\skills\gatling-fixer\SKILL.md)
4. [$gatling-scenario-data-creator](C:\Users\prakash\.codex\skills\gatling-scenario-data-creator\SKILL.md) for large-scale-ready scenario data

## Inputs
- Recording/scenario identifier
- Target alias (default: `ablfhir`)
- Fixer controls: `MaxIterations` (default `20`), optional `AutoApproveChanges`
- Scenario-data controls: `UsernameBase`, `PersonLast`, optional `DbEnv` (`fpabl` default, `ablfhir` optional)

## Workflow
1. Run converter to generate local Gatling script folders under `NBS/gatling/script/<scenario-name>`.
2. Immediately run gatling-runner once to validate baseline execution and generate failure diagnostics.
3. Run gatling-fixer to iteratively stabilize script transactions from report evidence.
4. Re-run gatling-runner after fixer completes to confirm stabilization.
5. Run scenario-data-creator to regenerate `globalDataSets` for scaled execution.
6. Use the scenario-data-creator final runner execution as the post-data-generation validation run.

## Execution Rules
- Run foreground by default.
- Stop only on hard blockers (missing files, auth failure, unreachable remote host, or fatal script errors).
- Preserve backups/reports produced by fixer and scenario-data-creator.
- Keep all reports under `C:/Users/prakash/Desktop/project/NBS/gatling/reports/<scenario-name>/`.

## Command Sequence (Reference)
```powershell
# 1) Convert
pwsh -NoProfile -Command "& 'C:/Users/prakash/.codex/skills/gatling-converter/scripts/run_skill_script.ps1' -ScriptName 'run_gatling_converter.ps1' -- -RecordingPath '<recording-folder>' -TargetAlias 'ablfhir'"

# 2) Validate run
pwsh -NoProfile -Command "& 'C:/Users/prakash/.codex/skills/gatling-runner/scripts/run_skill_script.ps1' -ScriptName 'run_gatling_remote.ps1' -- -ScenarioName '<scenario-name>' -TargetAlias 'ablfhir'"

# 3) Auto-fix
pwsh -NoProfile -Command "& 'C:/Users/prakash/.codex/skills/gatling-fixer/scripts/run_skill_script.ps1' -ScriptName 'run_gatling_script_auto_fix.ps1' -- -ScenarioName '<scenario-name>' -TargetAlias 'ablfhir' -MaxIterations 20"

# 4) Validate after fix
pwsh -NoProfile -Command "& 'C:/Users/prakash/.codex/skills/gatling-runner/scripts/run_skill_script.ps1' -ScriptName 'run_gatling_remote.ps1' -- -ScenarioName '<scenario-name>' -TargetAlias 'ablfhir'"

# 5) Refresh scenario-data for large scale (includes final run)
pwsh -NoProfile -File "C:/Users/prakash/.codex/skills/gatling-scenario-data-creator/scripts/run_gatling_scenario_data_creator.ps1" -ScenarioName "<scenario-name>" -UsernameBase "<username-base>" -PersonLast "<person-last-prefix>" -DbEnv fpabl -TargetAlias ablfhir
```

## Output Expectations
- Converted script present in `NBS/gatling/script/<scenario-name>`.
- Baseline and post-fix validation reports generated by runner.
- Fix iteration log in `<scenario-name>-autofix-report.html`.
- Updated `scenario-data.yaml` `globalDataSets` with backup created before replacement.
